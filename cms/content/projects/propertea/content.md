It is a site to aggregate all of the properties you are interested in, regardless of where they came from. The web app is built to be as collaborative as you want. I avoided any account sign-up stuff, partially for time-sake and partially because there are already so many accounts a person has to create when looking for a flat.

![Gallery image 1](/cms-assets/projects/propertea/gallery-1.png)

You are able to assign each listing a status to help keep track of where you are in the application process for each.

![Gallery image 2](/cms-assets/projects/propertea/gallery-2.png)

Adding a new listing is handled through dynamic web scraping (e.g. it can handle any property URL, even if it’s some poorly formatted Facebook post). This uses GPT-4 under the hood and I found it to be extremely powerful! This web scraping feature makes adding properties as fast as possible (under 20 seconds). It was important to ensure the process wasn’t cumbersome, as that would immediately disqualify this as a useful tool in the flat hunting process.

![Gallery image 3](/cms-assets/projects/propertea/gallery-3.png)

After just one post in a group of students looking for accommodation, the app gained 90 users in its first week. While the student flat-hunting rush has slowed, and I haven’t marketed the app further, it remains a valuable tool in hibernation. I plan to refine it, possibly add some ads, and relaunch it next summer when the next wave of students begins their search.
